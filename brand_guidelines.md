# ProofDive — Aesthetic Guidelines (v1.1)

## 1. Visual Philosophy
Minimal. Neutral. Precise.
- System over brand.
- Restraint over expression.
- Clarity over creativity.

## 2. Colour System — Base
- **Background**: `#0A0A0A`
- **Primary Text**: `#FFFFFF`
- **Secondary Text**: `#9CA3AF`
- **Dividers**: `#1F2937`

## 2. Colour System — Status
- **Not Ready**: `#DC2626`
- **Borderline**: `#6B7280`
- **Ready**: `#EAB308`
- **Star**: `#22D3EE`
- **Role Model**: Gold (use gradient, not fixed HEX)

### Colour Rules
- Colour is used only for meaning (status).
- No decorative use of colour.
- One dominant signal per screen.
- UI should remain ~90% neutral.

## 3. Typography
- **Font**: Urbanist
- Single font system only.
- No decorative styles.
- Maintain high readability.

## 4. Layout & Spacing
- Grid-based structure.
- Generous spacing.
- Avoid crowding.
- Left-aligned text preferred.

## 5. UI Style
- Flat, clean, structured.
- Minimal shadows.
- No gradients (except controlled cases).
- Functional, not decorative.

## 6. Iconography
- Minimal or no icons.
- If used: simple, line-based, neutral.

## 7. Motion
- Subtle, fast, purpose-driven.
- No exaggerated or playful animation.

## 8. Aesthetic References
- **Linear** — minimal UI, spacing, typography + subtle animations.
- **Palantir** — seriousness, tone + subtle animations.
- **Whoop** — performance & scoring visuals + subtle animations.

## Summary
ProofDive uses a restrained, minimal visual system where typography, spacing, and colour signals present information with clarity and authority.
